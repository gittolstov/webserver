В прошлом дне я рассказал, как сделать сервер. Здесь я покажу, что он может выдавать, как страницу.
Если мы используем render_template, то мы cможем использовать разные файлы, как шаблоны.
В примере используется файл "login.html"
Здесть я приведу его краткий код:
{ % extends "base.html" %} - кусок кода на ginger, означает, что наш файл выдаст шаблонное содержание base.html
ginger позволяет создать файл-шаблон (base.html), содержащий блоки
{ % block generic_block_name %}
*your html-code*
{ % endblock %}
Такой блок в файле-шаблоне можно заменить в последующий файлах, использующих ваш шаблон, таких, как login.html
Для изменения блока относительно шаблона используем такую же нотацию блока, и он перезапишет блок со своим именем из шаблона.
{ % block generic_block_name %}
*your modified html-code*
{ % endblock %}
При выдавании этого файла (login.html) через render_template, сервер выдаст HTML-код из base.html,
но блок "generic_block_name" будет заменён на такой в login.html
Сотрите пробелы после первых фиигурных скобок.
